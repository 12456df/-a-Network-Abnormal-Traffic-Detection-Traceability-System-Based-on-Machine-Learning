一、需求分析（实际系统能力）
--------------------------------------------------------------------------------

1. 数据来源与格式
   - 主路径：公开数据集 CIC-IDS2017 的流级 CSV（默认目录 rowdata/CSVs/TrafficLabelling，
     与 MachineLearningCVE 同类标注数据；由 csv_preprocessing.py 合并、清洗、二分类
     标签对齐与训练/验证/测试划分）。
   - 扩展路径：原始 PCAP → 流特征 CSV。cicflowmeter_preprocessing.py 调用 CICFlowMeter
     生成与 CIC 体系一致的 flow 特征列（另见 requirements-pcap.txt、
     docs/PCAP_MANUAL_SETUP_zh.txt）。
   - 开题中提到的 UNSW-NB15、CTU-13、NetFlow 等：可作为后续扩展数据源；当前主流程
     以 CIC-IDS2017 风格 CSV 为基准实现。

2. 特征与标签
   - 流级特征：直接采用数据集中已有数值特征列（及经预处理后的可用列），二分类标签
     由「Label 是否为 BENIGN」映射为 binary_label（0/1）。
   - 无监督侧：在 unsupervised_model.py 中支持基于训练集拟合 StandardScaler、可选 PCA
     降维、可选互信息（mutual_info_classif）做特征筛选，再输入 Isolation Forest 与
     自编码器族模型。
   - 开题中单独强调的「包级滑窗时序切片、会话深度手工特征工程流水线」：未单独拆
     模块；CICFlowMeter/TrafficLabelling 特征本身已包含时序与统计类 flow 属性，与本
     题「流表机器学习检测」目标一致。

3. 检测与融合
   - 有监督：RandomForestClassifier（supervised_model.py），含验证集上双阈值策略
     （高精确 / 高召回）及测试集 flow 级输出，供后续集成与溯源关联。
   - 无监督：Isolation Forest + 全连接 Autoencoder + VAE（PyTorch，unsupervised_model.py），
     支持 CPU/CUDA。
   - 集成：ensemble_detector.py 将 RF 与无监督分数级联，定义四级告警（CRITICAL/
     HIGH/MEDIUM/LOW），并导出集成预测与 IP 级汇总等，供溯源与可视化使用。
   - 开题中的 XGBoost、LSTM/1D-CNN：未纳入当前主 requirements.txt 与训练主路径；若论
     文需要对比，可作为展望或补充实验另建分支实现。

4. 溯源与可视化
   - attack_traceback.py：基于集成检测结果与元数据，完成（1）可疑源 IP 威胁评分排序；
     （2）利用 NetworkX 构建通信关系并做路径/子图层面的分析；（3）证据汇总与报告
     JSON；（4）静态图（时间线等）及可选 pyvis 交互图 HTML。
   - visualization.py：对监督/无监督/集成输出做指标曲线、混淆矩阵、PCA 等分析图。
   - 开题中的 GNN 传播评分：当前未使用图神经网络；图结构分析以 NetworkX + 规则/统计
     评分为主，与开题「轻量级、可部署」取向一致，实现路径更简化。

5. 非功能需求（与开题对齐部分）
   - 模块化脚本：预处理、训练、集成、溯源、可视化相互独立，可顺序复现实验。
   - 可复现：随机种子、JSON/CSV 结果落盘；模型 joblib/torch.save 保存。
   - 开题指标（如 F1≥0.85、延迟≤200 ms、1000 flows/s 等）：需在具体硬件与数据子集上
     实测后填入论文「实验与评估」章节；本文档不代测数值。

二、理论与技术方法（实际采用）
--------------------------------------------------------------------------------

1. 理论基础
   - 有监督二分类：基于标注流量区分 BENIGN / ATTACK，阈值优化兼顾运维误报与安全漏报。
   - 无监督异常检测：Isolation Forest 基于随机分割的异常得分；自编码器/VAE 基于重构
     误差（及 VAE 隐空间正则）刻画偏离正常流分布的样本。
   - 集成决策：多模型分数联合与分级告警，弱化单模型在边界样本上的不稳定性。
   - 溯源：将 flow 级告警聚合到 IP/边，结合图论中的子图与时间序关系做可疑源排序与
     路径展示（非 GNN 的显式推理链）。

2. 算法与工具清单（与代码一致）
   - 机器学习：scikit-learn（RandomForest、IsolationForest、train_test_split、部分
     特征选择与度量）。
   - 深度学习：PyTorch（Autoencoder、VAE、DataLoader）。
   - 图分析：networkx（attack_traceback.py）。
   - 数据处理：numpy、pandas、joblib。
   - 可视化：matplotlib、seaborn；可选 pyvis（交互拓扑）。
   - PCAP 工具链（可选环境）：CICFlowMeter、scapy（见 requirements-pcap.txt）等。

三、软硬件开发平台参数（实际采用）
--------------------------------------------------------------------------------

1. 软件平台
   - 操作系统：Windows 10/11（64 位）为主（与本项目 PCAP/CICFlowMeter 文档一致）；Linux
     亦可运行主流程（Python 脚本无平台独占逻辑，仅路径与外部工具安装不同）。
   - 编程语言：Python 3.10～3.12（主依赖见项目根目录 requirements.txt；PCAP 方案建议
     3.12+ 与 cicflowmeter 版本要求一致）。
   - 核心依赖（requirements.txt）：numpy、pandas、scikit-learn、joblib、matplotlib、
     seaborn、torch。
   - 溯源模块额外依赖：networkx；交互图需可选安装 pyvis。
   - 版本控制：Git（与开题一致）。

2. 硬件平台
   - 详见 docs/毕业设计软硬件配置.txt（最低/推荐配置、CPU 内存磁盘、可选 NVIDIA
     GPU 与 PyTorch CUDA 等）。

3. 与开题 2.2「软硬件平台」对照
   - 一致部分：Windows 11、Python 3.12（推荐）、Pandas/NumPy、scikit-learn、PyTorch、
     NetworkX、Matplotlib、Git。
   - 调整部分：主工程未固定引入 scipy/xgboost；scipy 出现在 requirements-pcap.txt；
     scapy 主要在 PCAP 虚拟环境配合 CICFlowMeter；主检测流水线不依赖 dpkt。

四、技术路线（实际实施步骤）
--------------------------------------------------------------------------------

以下顺序与仓库脚本设计一致，可对应论文「系统设计与实现」或「技术路线」插图。

  步骤 1 — 数据准备
    · 获取 CIC-IDS2017 等 CSV（或 PCAP 经 CICFlowMeter 转为 CSV）。
    · 运行 csv_preprocessing.py：合并、清洗、binary_label、划分 train/val/test，
      写出 processed/ 下特征表与 trace_metadata 等。

  步骤 2 — 有监督训练
    · 运行 supervised_model.py：训练 RandomForest，确定阈值，输出模型与测试集预测/
      指标及 flow 级异常标记文件。

  步骤 3 — 无监督训练
    · 运行 unsupervised_model.py：特征缩放与可选筛选/降维，训练 Isolation Forest、
      Autoencoder、VAE，保存权重与验证/测试得分。

  步骤 4 — 集成检测
    · 运行 ensemble_detector.py：加载 RF 与无监督模型，产生四级告警与集成预测、
      IP 汇总等，写入 processed/ 供溯源使用。

  步骤 5 — 攻击溯源
    · 运行 attack_traceback.py：读取集成输出与 trace 元数据，生成 IP 排名、路径/
      子图结构数据、HTML/PNG 与 summary JSON。

  步骤 6 — 可视化与论文图表
    · 运行 visualization.py（及溯源模块自带图）：生成 ROC、PR、混淆矩阵、PCA 等。

  步骤 7 — 评估与撰写
    · 在测试集上统计 Precision、Recall、F1、AUC 等（开题所列指标可在此步骤实测填表）。
    · 溯源部分重点报告 Top-K 可疑 IP 与路径可视化效果，若与开题「路径重建准确率」
      定义不同，需在论文中给出本实现的评价口径。

--------------------------------------------------------------------------------
五、与开题「拟采取方案」的差异小结（便于论文诚实表述）
--------------------------------------------------------------------------------

  · 有监督侧：实现为 RandomForest 主模型；XGBoost 可作为今后对比实验扩展。
  · 无监督侧：已实现 Isolation Forest、Autoencoder、VAE；未实现开题中的 LSTM/1D-CNN
    时序深度模型。
  · 溯源侧：采用 NetworkX + 统计/规则评分与可视化；未实现 GNN。
  · 数据集：主实现围绕 CIC-IDS2017 风格 CSV；UNSW-NB15/CTU-13 需格式适配后可接同一
    预处理接口扩展。
